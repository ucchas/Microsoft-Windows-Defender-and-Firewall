Enabling and Configuring Windows Firewall

OS: Windows 10

Objective:

  • Configure firewall rules using Windows Defender firewall
  • Configure firewall rules using Windows Defender Firewall with advanced security

Configure Firewall rules using Windows Defender Firewall

Windows security > Firewall & network protection

Domain network: Domain networks are workplace networks. A computer must be a part of the domain in order to communicate with other computers on that network.

Private network: Private networks are discoverable networks, meaning that only devices on that network can see or discover other devices on that same network. Home networks are a good example of a private network.

Public network: Public networks are non-discoverable networks. A non-discoverable network is a network where your device cannot be discovered by other devices on your network. A coffee shop or a library would be a good example of a public network. You do not want other individuals to be able to discover your device.

The Domain Network button is selected.

The Windows Defender Firewall option needs to be toggled On. The checkbox under Incoming connections is a quick access location that blocks all incoming domain network traffic - even traffic that is normally permitted.

The Private network button is selected.

Again here, the Windows Defender Firewall needs to be toggled On. To block all incoming private network traffic - even traffic that is normally permitted, the quick access checkbox under Incoming connections can be levereged.

Similarly, in the Public network section, it is to be verified that the Windows Defender Firewall is toggled On. To block all incoming public network traffic - even traffic that is normally permitted, the quick access checkbox under Incoming connections is to be noted.

On the Firewall & network protection Window, clicked Allow an app through firewall. An Allowed apps window is opened. Mozilla Firefox is found in the list of apps.

The current configuration allows for Firefox to communicate on the Private network but denies it from communicating on the Public network. The Public box is checked and clicked OK. Now users will be able to use Mozilla Firefox on the public network.

Configure Firewall rules using Windows Defender Firewall with Advanced Security

Objective:

  • To use Windows Defender Firewall with Advanced Security to edit an existing firewall rule
  • Allow the connection for Key Management Service on the Domain and Private network
  • Deny the connection for Key Management Service on the Public network

Windows Security > Firewall and network protection > Advanced settings

An Overview is presented in the center panel. Special note of the two rule types listed on the left panel:

  • Inbound rules: Determines what traffic is allowed to the computer
  • Outbound rules: Determines what traffic is allowed to leave the computer
  • Connection security rules
  • Monitoring

Each of the rules can be configured to filter traffic based on computers, users, applications, ports, protocols, and so on.

Clicked on Inbound rules.

A long list of inbound rules is shown on the Overview panel. Rules with a green checkmark next to them indicate that the rule is enabled to allow inbound communication. The rules without the checkmark are available for use but are not enabled.

Key Management Service is selected from the list. It is to be noted that:

  • The policy is currently not enabled
  • If enabled, the rule would allow communication

By double-clicking on the rule, the properties window is opened.

Here in the General tab, the name of the rule, a description of the rule, and whether the rule has been allowed or blocked is shown. The action is set to be allowed in this case. Now we go to the Advanced tab.

The Key Management Service rule applies to Domain, Private, and Public profiles (All), as all are selected. But because we want to allow communication only with the domain and private networks, for Public this box should not have a checkmark. Hence, the checkmark from Public is removed and clicked OK.

At this stage, a new inbound rule is to be created that blocks communication with the public network. Since the new rule will be similar to the last, the existing rule is copied and pasted onto the same overview panel. Now we select the second Key Management Service (TCP-In) inbound rule and double-click to open properties.

Since we want to block connection with the public network, Block the connection is selected on the General tab and clicked Apply.

Now on the Advanced tab the Domain and Private box checkmarks are removed and the Public box checkmark is enabled and clicked OK.

The Overview panel is now showing the changes. Now the rule is to be enabled by right-clicking on the rule and clicking Enable rule.

Now a green checkmark appears next to the first rule indicating that the rule allowing communication is enabled. A circle with a line through it appears next to the second rule indicating that the rule blocking communication is also enabled.

Configure firewall rule exercise

Objective:

  • To use Windows Defender Firewall with Advanced Security to block Windows Remote Management on the Public network

Windows Security > FIrewall and network protection > Advanced settings

In the Inbound rules, found two rules named Windows Remote Management (HTTP-In). One is for the domain network and the other is for the public network.

Double-clicked and entered into the properties of Windows Remote Management (HTTP-In) rule for the public network.

Under the General tab, selected Block the connection and then clicked OK.

Now we can see that the Windows Remote Management rule blocking connection on the public network is enabled.
